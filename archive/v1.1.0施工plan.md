# LinoI v1.1.0 施工 Plan（已完成，2026-07-10）

> 归档自 PROJECT_PLAN.md。

## 当前 Plan（v1.1.0 · 中型升级）

目标：把写作/提取链从 SSE 流式改为「后台任务 + 轮询」并把任务状态持久化；放宽字数区间、重设计记忆预算、修复短名误命中并加章级豁免；新增 LLM 调用审计表、单条人物事件接口、梗概/大事记可编辑、阅读模式等 App 体验项。单人 App 前后端同步发版，旧接口直接删，无兼容期（`chapter_style` wire 兼容仍保留）。

### 明确不做（防走样）
- 不做逐 beat 写作，不动「单次全章生成 + Reviser ≤2 次」链路。
- 不做章间连续性注入、不做全书大纲。
- 不动 Memory Selector 的 Agent 决策模式与预筛阈值（`prefilter_memory_candidates` 300 条/30000 字阈值不改）。
- `chapter_style` 兼容窗口本轮不收口（`_AuthorNoteCompat`、`ChapterRead.chapter_style` 镜像、iOS legacy 解码保留）。
- 不上向量库/embedding。

### 硬约束（对齐 CLAUDE.md 铁律）
- 新表/新列一律走 Alembic 迁移（SQLite 用 batch），发版前先备份云端 `linoi.db`（命令见 hk_info.md §13）。
- 字数按去空白字符数；程序校验优先于 Prompt，模型自报不作数。
- 审计表与日志绝不落 API Key、prompt、正文。
- 推理参数仍由 capability registry 决定，未知模型不发额外参数（本轮不动）。
- 后端测试须覆盖新行为并全量通过（现有 31 个为基线）；iOS 必须 `xcodebuild` App target 构建验证。

### 集中常量（全部落在 `Backend/app/services/context.py`）
- `WORD_COUNT_MIN_RATIO = 0.80`、`WORD_COUNT_MAX_RATIO = 1.20`；`word_count_bounds(target) -> (low, high)` 返回 `int(target*0.80), int(target*1.20)`。draft_violations、writer/reviser 契约文案统一取此函数，删掉散落的 `*0.95 / *1.05`。
- `MEMORY_BUDGET_CHARS = 1800`（固定，与 Bible 长度脱钩，可调）；`memory_budget()` 直接返回它（保留无参/忽略参数签名，改 chapters.py 调用点）。
- `MEMORY_SUMMARY_MAX_ITEMS = 2`（summary 类型装箱上限）。
- `CHARACTER_EVENT_MAX_CHARS = 60`（人物事件去空白字符上限，可调）；`truncate_to_nonspace(text, n)` 硬截断辅助。

### B1 后端：去流式 + 任务机制与持久化（`write_jobs.py`、`routers/chapters.py`、`schemas/chapter.py`、`main.py`）
- 删除 SSE：`stream_job`、`sse_event`、`_phase_event`、`WriteJob` 的 `buffer`/`condition`/`notify`/token 快照逻辑；删 `GET /chapters/{id}/write/stream` 接口与 `reattach_write`；`writer` 仍用 `complete_stream` 累积文本（保留 cancel 及时性），但不再对外发 token。
- `POST /chapters/{id}/write`：保持同步预检（`validate_character_preflight` 失败仍 409）；随后新建 `job_runs` 行（kind=write, phase=selecting_memory）→ reserve 内存 `WriteJob`（持 job_id + thread + cancel_event）→ `chapter.status="writing"` → 启动线程 → **立即返回 `WriteJobStatus`**（200）。
- 新增 `GET /chapters/{id}/job` → `WriteJobStatus`：有 `job_runs` 最新行则返回其快照；无行且 status 非 writing/extracting → phase=`idle`。worker 每次阶段切换 commit 到 `job_runs`（独立于最终章节事务），端点各自 `get_db` 会话读已提交值。
- worker `_run_job`：阶段推进写 `job_runs`（selecting_memory→writing→revising[attempt]→done/failed）；done 时把已校验正文落 `chapter.draft_text`+`status="draft_ready"`，failed（含 `revision_failed`）时 `_restore_baseline` 并写 phase=failed + error_code/message/violations。字数校验改用 `word_count_bounds`。
- `POST /chapters/{id}/write/cancel` 保留：取消内存 job、`job_runs` 置 cancelled、恢复 baseline、返回 `ChapterRead`。
- `WriteJobStatus`（`schemas/chapter.py`）：`chapter_id`、`kind`、`phase`、`attempt:int?`、`error_code:str?`、`error_message:str?`、`violations:list[dict]?`、`chapter:ChapterRead?`（done 时填）、`updated_character_ids:list?`、`added_event_ids:list?`（extract done 时填）。
- `main.py`：`recover_interrupted_chapters` 扩展——把所有非终态 `job_runs` 行标 `failed`（error_code=`interrupted`），章节 status 恢复逻辑不变；FastAPI `version`、`/health` 版本串改 `1.1.0`。
- 验收：`POST /write` 立即返回 phase=selecting_memory；轮询 `GET /job` 能走到 done 并带 finalized-前 draft_ready 章节；两次修订仍不合格时 `GET /job` 返回 phase=failed + code=`revision_failed` + violations，且旧 draft 不被覆盖；重启后非终态任务查得到 failed。

### B2 后端：accept/Extractor 异步化（`routers/chapters.py`、`write_jobs.py`）
- `POST /chapters/{id}/accept`：仍拒绝并发 live job；在 router 内用 `get_extractor_client` 建 `ExtractorAgent` 传入任务；新建 `job_runs`(kind=extract, phase=extracting) → `chapter.status="extracting"` → 启动后台提取线程 → **立即返回 `WriteJobStatus`**（phase=extracting）。
- 后台 `_run_extract_job`：跑 `extractor.extract` + `apply_extractor_output` + commit → phase=done（chapter finalized，填 updated_character_ids/added_event_ids）；失败恢复 `chapter.status="draft_ready"` 并 phase=failed（保留现有 `draft_ready` 恢复语义与 LLMError 分类）。复用同一 registry（按 chapter_id 防并发，`get_live` 对 write/extract 通用）。
- 前端轮询 `GET /job`（kind=extract）到 done=finalized。
- 验收：accept 立即返回 extracting；轮询到 done 时 chapter=finalized 且事件已覆盖写入；提取失败时章节回 draft_ready 且 `GET /job` 报 failed；重新 accept 覆盖本章旧事件（沿用现有事务覆盖）。

### B3 后端：字数放宽 + 记忆预算重设计 + 短名校验 + 章级豁免（`context.py`、`schemas/chapter.py`、`routers/chapters.py`）
- 字数：全部改用 `word_count_bounds`（80%~120%），含 `draft_violations`、`writer_user_message`、`reviser_user_message` 契约文案。
- 记忆预算：`memory_budget()` 返回 `MEMORY_BUDGET_CHARS`；`MemoryBlock` 加 `memory_type: str`（`headline`/`summary`/`character_event`），`memory_candidates` 构造时填。`pack_selected_memories` 在总预算内额外硬性执行 summary≤`MEMORY_SUMMARY_MAX_ITEMS`（超出的 summary 跳过，headline/character_event 不限条数）；`memory_selector_user_message` 提示「章节梗概最多选 2 条」。
- 短名校验 `scan_known_character_names`：≥3 字维持子串；**1 字名改左边界启发式**——在最长匹配游走中，命中 1 字名时仅当前一字符是串首或非汉字（`一-鿿` 之外，含空白/标点/英数/引号）才计入，否则视为词内（如「森林」不再命中「林」），拒绝时游标前进 1。**决策/风险**：2 字名维持子串匹配——若对 2 字名也强制边界，会漏掉「对林夕说」这类高频形式，得不偿失；残留的 2 字/前缀合成误命中由本轮新增「章级豁免」兜底，更精确的整词匹配见 Backlog 可选分词方案。（此点偏离需求「1~2 字都做边界」，请 build 前与用户确认或按此实现。）
- 章级豁免：`validate_character_preflight` 与 `draft_violations` 在得出 `unselected` 后减去 `chapter.exempted_character_names`（仅作用于「未授权人物」，`ambiguous_character_name` 仍硬报）。`ChapterPatch` 加 `exempted_character_names: list[str] | None`（None 不动），`ChapterRead` 输出该字段，`patch_chapter` 支持写入。
- 验收：目标 100 时 80 字过、79 字报 word_count；`scan("森林", 林)` 无命中、`scan("林进入", 林)` 命中、`林夕` 子串匹配与最长匹配不受影响（现有 preflight/重名测试仍绿）；把名字写进 `exempted_character_names` 后预检与复检都放行；summary 装箱最多 2 条。

### B4 后端：LLM 审计表 + 小接口 + 事件字数上限（`services/audit.py`(新)、`llm/openai_compatible.py`、`agents/*`、`routers/characters.py`、`services/extraction.py`、`schemas/character.py`）
- 审计接入：`OpenAICompatibleClient` 加 `last_usage: dict|None`（prompt/completion/total tokens，每次调用前清空）——非流式从 `data["usage"]` 取；流式 payload 加 `stream_options={"include_usage": True}` 并解析带 `usage` 的末块。`services/audit.py::record_llm_call(session_factory, *, agent_role, client, duration_ms, error_code=None, chapter_id=None, job_id=None)` 用**独立短会话** add+commit 一行 `llm_call_audits`（与 worker 事务解耦，rollback 不影响审计）。
- 统一接入点：write worker 的 memory_selector.select / writer.stream / reviser.revise、extract worker 的 extractor.extract 四处，各自计时并在成功/`LLMError` 两路都记一行（错误路记 error_code、无 usage 也可）。防御性 `getattr(client, "last_usage", None)`，兼容测试里的 Fake 客户端。
- `PATCH /character-events/{id}`（body `CharacterEventPatch{event_text:str}`）：改文本，写入前 `truncate_to_nonspace(text, CHARACTER_EVENT_MAX_CHARS)`，返回 `CharacterEventRead`（含 chapter_index）。`DELETE /character-events/{id}` → 204。
- 事件字数上限：`extractor_user_message` 增「每条 event_text ≤ 60 去空白字符」约束段；`apply_extractor_output` 保存新事件前对 `event_text` 硬截断（旧事件不动）。
- 验收：一次 write 后 `llm_call_audits` 至少有 writer 行且无 key/正文列；`LLMError` 路径记 error_code；PATCH/DELETE 事件生效；超 60 字事件被截断。

### B5 后端：ChapterPatch 放开 summary/headline + 版本（`schemas/chapter.py`、`main.py`、`README.md`）

- `main.py` 版本、`README.md` 顶部与能力段更新为 1.1.0。

### B6 iOS：去流式轮询 + 豁免交互（`LinoAPI.swift`、`LinoModels.swift`、`LinoStores.swift`、`ChapterEditorViews.swift`）
- `APIClient`：删 `streamWrite`/`reattachWrite`/`stream`/`decodeStreamPayload`；新增 `startWrite(chapterId, replaceDraft) -> WriteJobStatus`(POST /write)、`jobStatus(chapterId) -> WriteJobStatus`(GET /job)、`accept(chapterId) -> WriteJobStatus`(POST /accept)；`cancelWrite` 保留。
- `LinoModels`：删 `StreamPayload`；加 `WriteJobStatus{chapterId,kind,phase,attempt?,errorCode?,errorMessage?,violations:[Violation]?,chapter:Chapter?,updatedCharacterIds?,addedEventIds?}`，`Violation{code,message,names?}`。`Chapter` 加 `exemptedCharacterNames:[String]`（decodeIfPresent 默认 []）。
- `ChapterEditorStore`：删 streamTask/activeStreamId/streamedDraft/generationBaseline 的 SSE 分支，改为 pollingTask（间隔 2.5s 调 `jobStatus` 到终态）。`WritingPhase` 改为 `idle / selectingMemory / writing / revising(attempt) / extracting / failed(code,message)`（去掉 token 级字数）。`generate()`：save→startWrite→轮询；done 用 `status.chapter` 刷新+`idle`，failed→恢复 baseline+`.failed`+`notices.publish(critical: code=="revision_failed")`。`accept()`：save→accept→轮询到 done（finalized）→reload characters→**自动进入阅读模式**（见 B7），failed 恢复 draft_ready。`load()`：status 为 writing→轮询 write、extracting→轮询 extract；`handleScenePhaseActive` 与冷启动按章节 status 重新起轮询（替换原 reattach）。
- 豁免：预检 409（detail.code=`unselected_characters_in_bible`）或任务 failed（code=`unselected_character` + violations[].names）时，store 暴露 `pendingExemptionNames` 与 `exemptAndRetry()`（PATCH 写 `exempted_character_names` 后重新 generate）；`ChapterPatchPayload` 加 `exempted_character_names`。编辑器在报未授权人物时给「本章豁免并重试」入口。
- UI：生成/提取期间显示转圈 + 阶段文案（正在选择记忆/正在写作/第 1|2 次修订/提取中）；无 token 流，正文在 done 时整块刷新。
- 验收：`xcodebuild` App target 通过；模拟器生成→轮询→done 正常，revision_failed 走 Toast（critical）；切后台回前台/冷启动能续轮询；豁免后重试可生成。
