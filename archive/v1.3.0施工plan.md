# LinoI v1.3.0 施工与验收记录

## 目标

- Reviser 工作时在 iOS/macOS 实时展示程序校验未通过的具体原因，最终 `revision_failed` 保留最后原因。
- Writer 获得紧邻上一已完成章节的原文结尾用于开场衔接，同时保持本章 Bible 的最高权威和 1800 字历史上下文总预算。
- 修复双端本地任务状态、`write_running` 接管、数字型上游错误码和带 Z 六位微秒时间解析。

## 实现契约

- Memory Selector 只看紧邻上一章最多 700 个去空白字符的结尾候选，返回 `memory_ids` 与可选 `previous_ending_start_id`；后端只按候选 ID 原样截取，非法或缺失 ID 确定性降级。
- 上一章结尾优先占用 1800 字预算，其他大事记、梗概和人物事件使用剩余预算；Writer Prompt 将历史标为只读且低于 Bible，禁止据此增加剧情或授权人物。
- Memory Selector 新协议由代码固定追加，既有生产 Persona 和用户自定义 Persona 不会阻断新字段。
- 客户端在 `revising` 聚合 `violations.message`；非 `revision_failed` 的 LLM/网络失败清除旧验证原因。
- `/job` 在进程内 live job 已保留、SQLite 新行尚未可见的竞态窗口返回当前非终态快照，客户端据此接管并轮询。
