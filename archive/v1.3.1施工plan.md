# LinoI v1.3.1 施工与验收记录

## 目标

- iOS/macOS 阅读模式切换章节后回到新章节开头。
- 本地草稿仅在真实内容变化时标脏，恢复提示只保留页内横幅。
- 字数不足与截断只由 Writer 扩写，超长及其他程序违规继续由 Reviser 修复。
- 正式识别 GLM 5 系列的思考、推理强度与 Temperature 参数。

## 实现契约

- 字数继续按去空白字符机械校验，目标区间保持 80%～120%。Writer 与 Reviser 各最多修复两次，每轮重新校验；任何需要增加篇幅的结果都回到 Writer。
- Writer 扩写复用现有 `writing + attempt + violations` 状态，不增加数据库字段或新 phase 字符串；旧客户端仍按普通 writing 处理。
- 正常 Writer Prompt 仅增加内部篇幅分配和防提前收束，不重复生产 Persona 已有的“只扩写、不创造”约束。
- GLM 5/5.1/5.2 发送显式 thinking 开关，开启时可发送 high/max effort；Temperature 保持可配置。

## 验收与部署

- 后端 75 tests passed，覆盖不足、截断、超长、混合违规、Reviser 压缩过头与双方次数耗尽。
- iOS Simulator Debug 与 macOS Debug 构建成功，版本均为 `1.3.1(10)`。
- 生产部署、macOS Release 换包与启动验证待完成。
